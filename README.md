# PRD: 커뮤니티 다마고치 버디 (Community Tamagotchi Buddy)

## Context
사내 해커톤(1일, 1~2명) 참여를 위한 기획안. 기존 앱의 커뮤니티 기능에 다마고치 스타일의 히든 이스터에그를 추가하여, 유저 간 협력과 커뮤니티 활성화를 유도하는 것이 목표.

---

## 1. 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | Community Tamagotchi Buddy |
| 목표 | 커뮤니티 내 유저 간 협력을 통해 히든 명령어를 발견하고, 다마고치를 키우는 재미 제공 |
| 기술 스택 | React Native + Expo (클라이언트 전용, 서버 불필요) |
| 기간 | 1일 해커톤 |
| 팀 규모 | 1~2명 |

### 네이티브 앱으로 빌드·설치 (포팅)

이 저장소는 **이미 모바일 앱(Expo / React Native)** 입니다. 웹 전용이 아니라 iOS·Android **설치 패키지**로 가져가려면 아래 중 하나를 쓰면 됩니다.

1. **Expo Go로 바로 테스트 (가장 빠름)**  
   - `npm run dev` (또는 `npx expo start`) 후 QR 코드로 **Expo Go** 앱에서 연다.  
   - 별도 빌드 없이 실기기에서 동작 확인 가능.

2. **EAS Build로 APK / IPA 만들기 (배포·시연용)**  
   - [EAS](https://docs.expo.dev/build/introduction/)에 가입한 뒤 터미널에서:
     - `npm i -g eas-cli` → `eas login` → 프로젝트 루트에서 `eas init` (최초 1회, `app.json`에 EAS `projectId` 연동)  
     - 안드로이드 내부 배포용 APK: `eas build --platform android --profile preview`  
     - iOS는 Apple 개발자 계정·인증서가 필요 (같은 명령에 `--platform ios`).  
   - `app.json`의 `ios.bundleIdentifier` / `android.package` 는 스토어·실제 출시 전 **본인 도메인에 맞게** 바꾸는 것을 권장합니다 (현재 예시: `com.tamagotchibuddy.app`).

3. **로컬에서 Xcode / Android Studio로 실행 (네이티브 프로젝트 생성)**  
   - `npx expo prebuild` 로 `ios/`, `android/` 폴더 생성 (저장소 `.gitignore`에 제외되어 있음).  
   - `npx expo run:ios` 또는 `npx expo run:android` 로 시뮬레이터·에뮬레이터 또는 USB 기기에 설치.

커스텀 네이티브 모듈을 쓸 계획이면 `expo-dev-client` 를 추가하고 `npm run dev:client` 로 개발 서버를 띄우는 흐름을 [Development builds](https://docs.expo.dev/develop/development-builds/introduction/) 문서대로 맞추면 됩니다.

## 3. 기능 상세

### 3-1. 히든 명령어 시스템

- **트리거 조건**: 특정 게임 설치 또는 N분간 플레이
- **힌트 제공**: 트리거 달성 시 유저에게 명령어의 일부 힌트를 노출
  - 힌트는 유저마다 다른 조각을 받음 (예: 유저A는 "첫 글자는 B", 유저B는 "세 번째 글자는 D")
  - 힌트 UI: 커뮤니티 화면 내 배너 또는 팝업
- **명령어 입력**: 커뮤니티 텍스트 필드에 조합된 명령어 입력 시 다마고치 페이지 전환
- **MVP 범위**: 하드코딩된 1개의 명령어 + 3~4가지 힌트 조각

### 3-2. 다마고치 페이지 UI

- **외형**: 다마고치 기기 모양의 프레임 (둥근 사각형, 레트로 스타일)
- **화면 내부**: 도트 스타일 배경 + 도트 캐릭터 애니메이션
- **캐릭터**: 도트(픽셀) 아트 스타일, idle 상태에서 미세한 움직임
- **하단 버튼 3개**:

| 버튼 | 기능 | 규칙 |
|------|------|------|
| 밥 주기 | 배고픔 해소 | 1시간 미급식 시 병걸림, 병 후 6시간 방치 시 사망 |
| 똥 치우기 | 똥 제거 | 30분마다 똥 1개 생성, 6개 누적 시 사망 |
| 놀아주기 | 행복도 증가 | 캐릭터가 웃으며 좌우로 흔들리는 애니메이션 |

### 3-3. 캐릭터 성장 시스템 (MVP)

```
알 (0~1시간) → 유년기 (1~6시간) → 성장기 (6시간~)
```

- 3단계 일직선 성장 (분기 없음, MVP)
- 각 단계별 다른 도트 스프라이트
- 성장 조건: 시간 경과 + 생존 상태

### 3-4. 상태 관리 (클라이언트 전용)

- **AsyncStorage** 또는 **MMKV**로 로컬 저장
- 저장 데이터:
  - `lastFeedTime`: 마지막 밥 준 시각
  - `lastCleanTime`: 마지막 똥 치운 시각
  - `poopCount`: 현재 똥 개수
  - `isSick`: 병 상태 여부
  - `sickSince`: 병 시작 시각
  - `isDead`: 사망 여부
  - `stage`: 현재 성장 단계 (egg / baby / grown)
  - `createdAt`: 다마고치 생성 시각
  - `isUnlocked`: 해금 여부
  - `battleRecord`: 전적 `{ wins: number, losses: number, npcWins: number }`
- 앱 포그라운드 복귀 시 경과 시간 계산하여 상태 갱신

### 3-5. 캐릭터 상태별 표시

| 상태 | 화면 표현 |
|------|----------|
| 정상 | idle 애니메이션 (미세한 위아래 바운스) |
| 배고픔 | 말풍선에 밥 아이콘 |
| 병 걸림 | 캐릭터 위에 해골 마크 + 파르르 떨림 |
| 똥 쌓임 | 캐릭터 옆에 똥 도트 아이콘 표시 (최대 6개) |
| 놀아주기 | 웃는 표정 + 좌우 흔들림 (2초간) |
| 사망 | 무덤 도트 이미지 + "다시 시작" 버튼 |

### 3-6. 대결 시스템

기존 위치 기반 공유 시스템을 활용하여 PvP/PvE 대결 기능 추가.

#### PvP (유저 vs 유저)
- **매칭 방식**: 커뮤니티 화면에서 다른 유저의 다마고치 프로필을 탭하여 대결 신청
- **대결 형태**: 터치/타이밍 기반 미니게임
  - 예시: 빠르게 탭하기, 타이밍 맞춰 버튼 누르기, 좌우 피하기 등
- **대결 조건**: 양쪽 다마고치 모두 생존 상태여야 대결 가능 (사망 상태 불가)

#### PvE (유저 vs NPC)
- **조우 방식**: 기존 위치 기반 시스템 활용, 특정 위치에 NPC 다마고치 배치
- **NPC 종류** (MVP):
  - 약한 NPC: 유년기 수준 — 초보 유저용
  - 강한 NPC: 성장기 수준 — 숙련 유저용
- **NPC 위치**: 하드코딩된 좌표 또는 특정 POI(관심지점) 기반

#### 미니게임 상세
```
┌─────────────────────────────┐
│  VS 대결!                    │
│                             │
│  (°▽°)     vs     (>_<)    │
│  내 버디          상대 버디  │
│                             │
│  ┌───────────────────────┐  │
│  │                       │  │
│  │    [ TAP! TAP! TAP! ] │  │
│  │                       │  │
│  │    ████████░░ 80%     │  │
│  │    ░░░░████████ 60%   │  │
│  │                       │  │
│  └───────────────────────┘  │
│                             │
│  제한시간: 10초              │
└─────────────────────────────┘
```
- 제한 시간 내 탭 횟수 대결 (MVP용 가장 단순한 형태)
- 다마고치 성장 단계에 따라 기본 스탯 보너스 부여
  - 알: 보너스 없음
  - 유년기: +10% 보너스
  - 성장기: +20% 보너스
- 건강 상태 페널티: 병 걸린 상태면 -30% 페널티

#### 대결 결과

| 결과 | 보상 |
|------|------|
| 승리 | 전적 기록 + 승리 카운트 증가 |
| 패배 | 전적 기록만 |
| NPC 승리 | 전적 기록 + 특별 승리 뱃지 (위치별) |

#### 전적 시스템
- 로컬 저장: `battleRecord: { wins: number, losses: number, npcWins: number }`
- 다마고치 화면에 전적 표시 (예: 5승 2패)

### 3-7. 젬 내기 시스템 (확장 기획)

> 해커톤 이후 확장 기능으로, MVP에는 포함하지 않음

- **내기 플로우**:
  1. 대결 신청 시 젬 베팅 금액 설정 (최소/최대 제한)
  2. 상대방이 동일 금액 수락
  3. 대결 진행
  4. 승자가 총 베팅 젬 획득 (수수료 10% 차감 가능)
- **안전장치**:
  - 1일 최대 베팅 횟수 제한 (예: 5회/일)
  - 최대 베팅 금액 상한선
  - 연패 시 쿨다운 타이머 (과도한 손실 방지)
- **UI**: 대결 신청 팝업에 젬 입력 필드 추가
- **서버 연동 필요**: 젬 트랜잭션은 서버에서 처리 (클라이언트 전용 불가)

## 5. 기술 구현 계획

### 필요 라이브러리 (Expo 호환)
- `react-native-reanimated` — 도트 캐릭터 애니메이션 (Expo SDK 내장)
- `expo-async-storage` (`@react-native-async-storage/async-storage`) — 상태 저장
- `expo-location` — 위치 기반 NPC 조우 (기존 위치 시스템 활용)
- `expo-haptics` — 버튼 터치/대결 시 햅틱 피드백
- `expo-router` — 커뮤니티 화면 → 다마고치/대결 화면 네비게이션
- 도트 스프라이트 이미지 — 직접 제작 또는 무료 에셋 활용

### 주요 컴포넌트 구조
```
TamagotchiScreen/
├── TamagotchiFrame       # 다마고치 기기 프레임 UI
├── PixelCharacter        # 도트 캐릭터 + 애니메이션
├── StatusIndicators      # 똥, 병, 배고픔 표시
├── ActionButtons         # 밥/청소/놀기 버튼 3개
├── GrowthStageLabel      # 현재 성장 단계 표시
├── BattleRecord          # 전적 표시 (5승 2패)
├── useTamagotchiState    # 상태 관리 커스텀 훅
└── useBattleState        # 대결 상태 관리 훅

BattleScreen/
├── BattleArena           # 대결 화면 (VS 레이아웃)
├── TapMiniGame           # 탭 미니게임 로직 + UI
├── BattleResult          # 승패 결과 팝업
└── NPCEncounter          # NPC 조우 화면 (위치 기반)
```

### 핵심 로직 (useTamagotchiState 훅)
```
- 앱 포그라운드 복귀 시:
  1. 경과 시간 계산
  2. 똥 개수 갱신 (경과시간 / 30분)
  3. 배고픔/병/사망 상태 판정
  4. 성장 단계 갱신
- 버튼 액션 시:
  1. 상태 업데이트
  2. AsyncStorage 저장
  3. 애니메이션 트리거
```

## 7. MVP 스코프 vs 확장

### MVP (해커톤 당일)
- [x] 다마고치 프레임 UI
- [x] 도트 캐릭터 3단계 (알/유년기/성장기)
- [x] 밥/청소/놀기 버튼 동작
- [x] 사망/부활 로직
- [x] 로컬 상태 저장
- [x] 하드코딩된 명령어 1개로 페이지 진입
- [x] PvP 대결 (탭 미니게임 + 전적 기록)
- [x] PvE NPC 대결 (위치 기반, 하드코딩 좌표 2~3개)

### 확장 (해커톤 이후)
- 젬 내기 시스템 (서버 연동 필요)
- 서버 연동으로 상태 영속화
- 분기 진화 시스템 (돌봄 패턴에 따라 다른 캐릭터)
- 커뮤니티 랭킹 (가장 오래 생존한 다마고치 / PvP 최다승)
- 다양한 명령어 + 힌트 세트
- 다양한 미니게임 종류 추가
- 다른 유저의 다마고치 방문
- NPC 위치 서버 관리 + 시즌/이벤트별 NPC 교체

## 8. 검증 방법

1. **기능 테스트**: 각 버튼 동작 확인, 시간 경과에 따른 상태 변화 확인
2. **시간 시뮬레이션**: 기기 시간을 앞으로 돌려 배고픔/똥/사망 로직 검증
3. **명령어 입력 테스트**: 커뮤니티 텍스트 필드에 올바른/잘못된 명령어 입력 시 분기 확인
4. **앱 재시작 테스트**: 앱 종료 후 재진입 시 상태 복원 확인
5. **데모 시나리오**: 알 부화 → 밥 주기 → 똥 치우기 → 놀아주기 → 성장 확인
6. **PvP 대결 테스트**: 대결 신청 → 미니게임 진행 → 승패 판정 → 전적 저장 확인
7. **NPC 대결 테스트**: 특정 위치 진입 → NPC 조우 → 대결 → 결과 확인
8. **성장 단계별 스탯 보너스**: 각 단계에서 대결 시 보너스 정상 적용 확인
